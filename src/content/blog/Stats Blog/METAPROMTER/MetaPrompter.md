**MetaPrompter: Definición, Misión, Visión y Servicio**

**¿Qué es MetaPrompter?**

**MetaPrompter** es un **sistema de ingeniería de software asistida por IA** que actúa como un **orquestador de decisiones** y un **generador de prompts quirúrgicos**. No es un simple formulario ni un generador de texto; es un **facilitador cognitivo** que traduce la visión difusa del usuario en especificaciones técnicas precisas, estructuradas y fundamentadas, listas para ser ejecutadas por agentes de IA (ChatGPT, Claude, Cursor, Copilot, etc.).

MetaPrompter integra tres corrientes de pensamiento:

1. **Clean Architecture** (bespoyasov): Para definir una estructura de código desacoplada, testeable y durable.
2. **Simplicidad Radical** (Ponytail): Para eliminar complejidad innecesaria y generar código eficiente y directo.
3. **Pedagogía Activa** (Juan David Velásquez): Para guiar al usuario paso a paso, construyendo conocimiento sólido y justificado.

MetaPrompter convierte el proceso de desarrollo en una experiencia de **aprendizaje y producción simultánea**.

**Misión**

Convertir la intención del usuario en código de calidad industrial, mediante un proceso estructurado que educa, guía y documenta cada decisión, asegurando que el software generado sea mantenible, escalable, seguro y estéticamente refinado.

**Componentes de la Misión:**

| **Elemento** | **Descripción** |
| --- | --- |
| **Intención → Especificación** | Capturar la visión del usuario y traducirla en requisitos concretos y no ambiguos. |
| **Educación continua** | Cada paso explica el *por qué* de las decisiones, formando al usuario en mejores prácticas. |
| **Documentación automática** | Generar ADRs (Architecture Decision Records) que justifiquen cada elección. |
| **Calidad industrial** | Asegurar que el código cumpla SOLID, testing, seguridad, accesibilidad y principios Tufte. |

**Visión**

Ser el estándar de referencia para el desarrollo de software asistido por IA, donde cada proyecto generado sea un modelo de excelencia en arquitectura, diseño y experiencia de usuario, y donde el proceso mismo sea una escuela de ingeniería para los equipos.

**Aspiraciones de la Visión:**

* **Estandarización global:** Que MetaPrompter se convierta en el "lenguaje común" para equipos de datos y desarrollo que utilizan IA.
* **Cultura de calidad:** Que el uso de MetaPrompter propague naturalmente principios como Clean Architecture, Tufte y SOLID.
* **Autonomía progresiva:** Que los usuarios, tras varias iteraciones, internalicen el método y puedan prescindir del asistente, pero elijan usarlo por su eficiencia.

**Servicio: El Flujo de Valor de MetaPrompter**

MetaPrompter ofrece un **servicio estructurado en 8 pasos** que cubre todo el ciclo de vida del software estadístico, desde la definición estratégica hasta el despliegue y la documentación. Cada paso es un **módulo de decisión** que acumula contexto y refina el prompt final.

**4.1. Diagrama del Servicio**

**4.2. Descripción de Cada Paso**

| **Paso** | **Propósito** | **Preguntas Clave** | **Referentes** |
| --- | --- | --- | --- |
| **0. Fundación** | Identidad y alcance | Nombre, dominio, madurez, equipo | Project Charter |
| **1. Estrategia** | Problema, usuarios, KPIs, compliance | ¿Quién usa el sistema? ¿Qué decisión tomará? ¿Qué mide el éxito? | Dashboards That Deliver, Juan David Velásquez |
| **2. Arquitectura** | Estilo y stack tecnológico | ¿Hexagonal o capas? ¿FastAPI o Express? ¿PostgreSQL o MongoDB? | Clean Architecture, bespoyasov |
| **3. Ingeniería** | Principios de calidad | ¿SOLID? ¿Tipado? ¿Manejo de errores con Result? | Clean Code, The Pragmatic Programmer |
| **4. Diseño UI/UX** | Estética, visualización, interacción | ¿shadcn/ui? ¿Recharts? ¿Principios Tufte? | Tufte, Cotgreave, Ponytail |
| **5. Seguridad** | Autenticación, autorización, cifrado | ¿JWT? ¿RBAC? ¿Vault? | OWASP, Software Engineering at Google |
| **6. Testing** | Estrategia de pruebas | ¿Unitarias? ¿Integración? ¿E2E? ¿Seguridad? | Test Driven Development, Clean Code |
| **7. Despliegue** | Docker, CI/CD, documentación | ¿Docker multi-stage? ¿GitHub Actions? ¿ADR? | full\_stack\_fastapi\_react\_template |

**4.3. El Artefacto Final: El Prompt Quirúrgico**

Al finalizar los 7 pasos, MetaPrompter genera un **prompt estructurado** que contiene:

1. **Contexto general:** Problema, usuarios, métrica de éxito.
2. **Arquitectura y stack:** Estilo, backend, frontend, base de datos, cola.
3. **Principios de ingeniería:** SOLID, tipado, inyección de dependencias, manejo de errores, logging.
4. **Diseño UI/UX:** Sistema de diseño, librería de gráficos, principios Tufte, jerarquía visual, accesibilidad.
5. **Seguridad y privacidad:** Autenticación, autorización, cifrado, logs de auditoría.
6. **Testing:** Cobertura, herramientas, integración en CI.
7. **Despliegue y documentación:** Docker, docker-compose, CI/CD, README, ADR.

Este prompt puede ser **copiado y pegado** directamente en cualquier agente de IA, garantizando que el código generado sea de calidad profesional desde la primera iteración.

**5. Valor Diferencial de MetaPrompter**

| **Atributo** | **Beneficio** |
| --- | --- |
| **Pedagógico** | No solo genera código, sino que educa al usuario en principios de ingeniería. |
| **Contextual** | Acumula y estructura la información del usuario, evitando prompts vagos. |
| **Referenciado** | Cada decisión se fundamenta en obras de maestros (Tufte, Martin, Velásquez). |
| **Integrador** | Combina arquitectura, diseño, seguridad, testing y despliegue en un solo flujo. |
| **Trazable** | Genera documentación de decisiones (ADR) automáticamente. |
| **Adaptable** | Ajusta el prompt según el dominio (finanzas, salud, retail) y la madurez del proyecto. |
